## 威胁模型与安全上下文

由于 **NTE Auto-Fishing** 是一个本地自动化运行的辅助工具，其在安全性上有以下特定的边界和考量：

### 1. 提权运行（管理员模式）
- **上下文**：在 Windows 平台上，本工具需要以管理员身份（Administrator）运行，才能使模拟输入（通过 `PyDirectInput` 和 `ctypes.windll`）被处于高权限运行的游戏窗口正确接收。
- **风险**：以管理员身份运行任何软件都会授予该软件对操作系统的完全控制权。如果工具的输入处理、GUI 解析或依赖包中存在漏洞，可能会被利用为本地提权（LPE）漏洞。
- **缓解措施**：我们尽可能减少依赖项，避免解析不可信的外部输入，并完全在本地运行。请始终确保直接从官方仓库 [Chizukuo/NTE-auto-fish](https://github.com/Chizukuo/NTE-auto-fish) 下载发布包。

### 2. 完全本地化运行（无网络通信）
- **上下文**：核心程序（包括 [main.py](../main.py) 和 [gui/](../gui/) 中的 DearPyGui 界面）完全在您的本地机器上执行。
- **隐私**：工具**不会**收集任何遥测数据，不会将屏幕捕获帧上传到网络，也不会与任何外部 Web 服务器进行通信。
- **例外**：仅在开发或安装阶段会使用标准的包管理器（`pip`）或 Git 工具下载更新。

### 3. 反作弊与账号安全
- **上下文**：虽然这是安全政策，但用户的账号安全对我们同样重要。
- **免责声明**：使用自动化工具、脚本或辅助软件可能违反游戏的《服务条款》（ToS），并可能导致账号被封禁或处罚。
- **缓解措施**：本工具使用了可配置的人性化鼠标和键盘操作机制，并完全依赖非入侵式的计算机视觉（OpenCV）技术，而非读写游戏内存。尽管如此，您仍需自行承担使用本软件带来的所有风险。

## 如何报告安全漏洞

> [!IMPORTANT]
> **请勿通过公开 Issue 的形式报告安全漏洞**。公开披露安全漏洞可能会在修复程序发布前，使用户的系统和软件暴露于风险之中。

如果您认为自己发现了本项目的安全漏洞，请按照以下步骤操作：

1. **提交私密漏洞报告**：
   - 访问本仓库在 GitHub 上的 **Security** 标签页。
   - 点击侧边栏中的 **Advisories**。
   - 点击 **Report a vulnerability** 以提交私密草稿公告。
   - 提供关于漏洞的详细描述，包括步骤明晰的重现方法、受影响的平台/操作系统信息，并尽可能提供一个可运行的漏洞概念验证（PoC）。

2. **备用联系方式**：
   - 如果私密漏洞报告功能不可用，您可以通过 GitHub 建立私密沟通渠道，或直接联系维护者 **Chizukuo**。

### 漏洞处理流程

- **确认接收**：我们将在 **48 小时** 内确认收到您的报告并对漏洞进行核实。
- **漏洞调查**：我们将与您紧密配合，复现并定位该安全隐患。
- **漏洞修复**：一旦确认，我们计划在 **7 至 14 天** 内发布修复补丁（视漏洞的修复难度而定）。
- **致谢**：在获得您的许可后，我们非常乐意在版本发布说明（Release Notes）及变更日志（Change Logs）中署名致谢您的贡献。
