# NTE Auto-Fish 設定ガイド

このガイドでは、NTE Auto-Fish の各設定項目の意味と調整方法について詳しく説明します。ゲーム環境やニーズに合わせて自動釣り Bot のパフォーマンスを最適化するのに役立ちます。

すべての設定は GUI 左側の **Settings** タブからリアルタイムで調整できます。変更後は必ず下部の `Save Settings` をクリックして保存してください。

---

## 1. PID Tuning (PID チューニング)
PID コントローラーは、釣りミニゲームでターゲットの安全ゾーンをスムーズかつ正確に追跡するために使用されます。カーソルが激しく揺れたり、追いつけなかったりする場合は、これらのパラメータを調整します。

*   **Kp (比例ゲイン):** 基本的な応答強度。**値が大きいほど、カーソルをターゲットに動かす力が強くなります**。カーソルが高速で動く安全ゾーンに追いつけない場合は、Kp を上げます。ターゲットの周りでカーソルが激しく前後に揺れる場合は、Kp が高すぎるので下げてください。
*   **Ki (積分ゲイン):** 長期的な定常偏差を排除するために使用されます。通常は小さな値 (例: 0.05) に保ちます。カーソルが常に中心からわずかにずれている場合は、Ki を少し上げます。
*   **Kd (微分ゲイン):** 振動を抑えるダンパーとして機能します。カーソルが常にオーバーシュートして前後に揺れる場合は、Kd を上げます。
*   **Deadband (デッドバンド/px):** カーソルと安全ゾーンの中心との距離がこの範囲内にある場合、Bot はキー操作を送信しません。**これにより、中心付近での過剰な修正による神経質な振動を効果的に防ぎます。** 推奨値は約 5.0 です。
*   **Integral limit (積分制限):** 積分項が蓄積しすぎて制御不能 (ワインドアップ) になるのを防ぎます。通常はデフォルトの 150 のままで問題ありません。
*   **Adaptive damping (適応ダンピング):** **有効化**を強くお勧めします。激しいカーソルの振動が検出された場合、システムは一時的に Kp を下げてカーソルを素早く安定させます。

## 2. Vision & Detection (視覚と検出)
このセクションでは、Bot が画面の色を介してゲーム要素をどのように識別するかを制御します。夕暮れや夜など、照明の変化に直面した場合は、ここで微調整を行う必要があるかもしれません。

*   **Safe Zone HSV (安全ゾーン HSV):** 釣りゲージ上の「ターゲット安全ゾーン」の色範囲。
*   **Cursor HSV (カーソル HSV):** 釣りゲージ上のプレイヤーの「カーソル」の色範囲。
*   **Bite Trigger HSV (アタリ検出 HSV):** 魚が食いついたかどうかを判断するために使用されるウキ/感嘆符の色範囲。
    *   *調整のヒント：HSV オプションを展開し、Min/Max スライダーを調整し、右側のプレビュー枠を見ながら、ゲーム内の実際の色にできるだけ近づけます。*
*   **Edge ignore ratio (エッジ無視比率):** 釣りゲージの左右の端を無視する比率 (例: 0.02 は左右の端から 2% を無視)。これにより、ゲージの枠をカーソルや安全ゾーンと誤認するのを防ぎます。
*   **Blue pixel trigger (青色ピクセルしきい値):** 「アタリ」を判定するために必要な、検出領域内の対象色のピクセル数。Bot が早く引きすぎる (誤検知) 場合はこの値を上げ、アタリを見逃す場合は下げてください。

## 3. Timing (タイミング)
Bot の状態遷移の各種遅延やタイムアウトを設定し、論理がスムーズに進行し、ソフトロックを防ぐようにします。

*   **Cast animation (s) (キャストアニメーション):** キャスト (竿を投げる) 後、アタリ検出を開始するまでの待機時間。早すぎると誤検知の原因になります。
*   **Bite timeout (s) (アタリタイムアウト):** キャスト後、アタリが来ない場合に自動的に再キャストするまでの最大待機時間。
*   **Lost frame limit (ロストフレーム上限):** 格闘中、カーソルまたは安全ゾーンがこのフレーム数連続して見失われた場合、格闘終了 (釣果または逃亡) と判定されます。
*   **Result wait (s) (リザルト待機時間):** 魚を捕獲した後、リザルト画面を閉じるまでの待機時間。ゲーム側が閉じる操作を認識しない場合は、長くしてください。
*   **Waiting poll (s) (待機ポーリング):** アタリを待っている間の画面確認頻度。
*   **Tracking poll (s) (追跡ポーリング):** 格闘中の PID と画像認識の更新間隔。低いほど反応は速くなりますが、CPU 使用率が上がります。
*   **Bait error limit (餌切れエラー上限):** キャストエラーが何回連続したらエラーダイアログをポップアップするか (通常は餌切れが原因)。無限ループを防ぐため、この上限に達すると Bot は停止します。
*   **Max struggle (s) (最大格闘時間):** 1回の格闘の最大許容時間 (デフォルト 120 秒)。バグが発生した場合のソフトロックを防ぎます。

## 4. Input & Hotkeys (入力とホットキー)
ゲーム内のキーバインドと Bot のグローバルホットキーを管理します。

*   **Key Bindings (キーバインド):**
    *   **Cast key:** キャスト/フッキングのキー (デフォルト: F)
    *   **Move left/right:** 格闘中に魚を引っ張る左右のキー (デフォルト: A および D)
    *   **Exit key:** UI 画面を閉じるキー (デフォルト: Esc)
*   **Result close (リザルト画面の閉じ方):** リザルトポップアップの閉じ方。「Click center」(中央クリック) または「Press exit key」(Exit キーを押す) を選択します。
*   **Always on top (常に最前面):** GUI ウィンドウをゲームウィンドウの常に前面に保ちます。
*   **Debug logging (デバッグログ):** 詳細な PID 追跡データを `fishing_data.csv` に書き込みます。問題のトラブルシューティング時にのみ推奨されます。
*   **Monitor (モニター):** 複数のモニターがある場合は、ゲームが実行されているモニターを選択して、キャプチャ領域が正しいことを確認します。
*   **Global Hotkeys (グローバルホットキー):**
    *   **Toggle:** Bot の一時停止/再開ホットキー (デフォルト: F8)。ゲームメニューを操作する前に使用することをお勧めします。
    *   **Stop:** Bot を完全に停止するホットキー (デフォルト: F12)。


## 5. Humanization (人間化/ヒューマナイゼーション)
ランダム性や不完全な反応を導入して、Bot の操作をより人間のプレイヤーに似せ、アンチチートに検出されるリスクを減らします。

*   **Humanize input (人間化を有効にする):** マスタースイッチ。常にオンにしておくことを強くお勧めします。
*   **Key Pulse Timing (キーパルスタイミング):**
    *   **Pulse hold min/max:** 格闘中にキーを押し続ける時間のランダムな範囲。
    *   **Pulse gap min/max:** キー入力間のランダムな遅延範囲。
*   **Deadband Micro-corrections (デッドバンド微調整):**
    *   **Deadband taps:** カーソルがデッドバンド内 (完璧に一致) にある場合、人間の手の震えや無意識の微調整を模倣するために、時折非常に短い「マイクロタップ」入力を送信します。
    *   **Tap chance:** ループごとにデッドバンドタップがトリガーされる確率。
*   **Reaction Latency (反応遅延):**
    *   **Reaction min/max:** 人間の視覚的反応時間をシミュレートします。Bot は動きを見てからこの時間待機してからキーを押します。
    *   **Reaction dist:** ランダム遅延の分布パターン (一様分布 uniform、ガウス分布 gaussian、指数分布 exponential)。
*   **PID Noise (PID ノイズ):** 
    *   完璧な PID 出力にランダムなノイズを重ね、移動軌跡を機械的でなくします。
*   **Timing Jitter (タイミングジッター):** 
    *   キャストアニメーションやリザルト待機などの固定時間にランダムな変動を追加します。
*   **Adaptive Focus (適応フォーカス):**
    *   実際のプレイヤーの行動をシミュレートします。魚が遠くにいて逃げそうなとき、プレイヤーの集中力は高まり反応が速くなります。有効にすると、誤差が大きいほど Bot の遅延が自動的に減少し、入力頻度が上がり、素早く魚を引き戻します。

## 6. System & Updates (システムとアップデート)

*   **Check for Updates:** GitHub で新しいリリースを確認します。見つかった場合は「Download Update」ボタンが表示され、ブラウザでリリースページを開くことができます。

---

**トラブルシューティング:**
1. カーソルが常に遅れる場合：`PID Tuning` の `Kp` が低すぎないか、または `Humanization` の `Reaction Latency` が高すぎないかを確認してください。
2. 夕暮れ/夜間に Bot が魚をフッキングしなくなる場合：ゲーム内の暖かい照明により青色ピクセルが減少します。`Vision & Detection` の `Blue pixel trigger` を少し下げるか、`Safe Zone` と `Cursor` の HSV 範囲を再校正してください。
3. 検出領域の調整が必要な場合：コンソールまたはメイン GUI ウィンドウから `recalibrate` を実行してください。
