# NTE Auto-Fish 设置指南

本指南将详细介绍 NTE Auto-Fish 中各个设置项的含义及调节方法，帮助您根据自己的游戏环境和需求，将自动钓鱼机器人的表现调整到最佳状态。

所有的设置都可以在图形界面 (GUI) 左侧的 **Settings** 选项卡中实时进行调整，并点击底部的 `Save Settings` 保存。

---

## 1. PID Tuning (PID 调节)
PID 控制器用于在钓鱼小游戏中平滑且精准地追踪目标安全区。如果发现指针抖动严重或者跟不上安全区，可以通过调节以下参数来优化：

*   **Kp (比例增益):** 基础的响应力度。**值越大，指针向目标移动的力度越大**。如果指针跟不上快速移动的安全区，可以适当调高 Kp；如果指针在目标附近剧烈左右抖动，说明 Kp 过高，请调低。
*   **Ki (积分增益):** 用于消除长时间的细微偏差。通常保持较小的值（如 0.05）。如果发现指针总是差一点点才能对齐中心，可以微微调高 Ki。
*   **Kd (微分增益):** 用于抑制震荡，起阻尼作用。如果指针总是冲过头（超调）并来回摆动，可以适当调高 Kd 以增加阻尼。
*   **Deadband (死区/像素):** 当指针与安全区中心的距离在这个范围内时，机器人不会发送任何按键操作。**这能有效防止指针在中心点附近因为过度微调而产生的神经质抖动**。推荐值 5.0 左右。
*   **Integral limit (积分限幅):** 防止积分项积累过大导致失控（积分饱和）。一般保持默认值 150 即可。
*   **Adaptive damping (自适应阻尼):** 强烈建议**开启**。当检测到指针发生剧烈震荡时，系统会自动临时降低 Kp 响应，使指针迅速稳定下来。

## 2. Vision & Detection (视觉与检测)
这部分控制机器人如何通过屏幕颜色来识别游戏元素。遇到夕阳/夜晚等光照变化时，可能需要在这里进行微调。

*   **Safe Zone HSV (安全区 HSV):** 钓鱼条上“目标安全区”的颜色范围。
*   **Cursor HSV (指针 HSV):** 钓鱼条上代表玩家位置的“游标指针”的颜色范围。
*   **Bite Trigger HSV (咬钩触发 HSV):** 鱼漂/感叹号/特定提示的颜色范围，用于判断鱼是否上钩。
    *   *调节建议：点击展开 HSV 选项，通过调节 Min/Max 滑块，观察右侧的颜色预览框，使其尽可能匹配游戏内实际的颜色。*
*   **Edge ignore ratio (边缘忽略比例):** 钓鱼条左右两端的忽略比例（例如 0.02 表示忽略左右各 2% 的区域）。用于防止将钓鱼条边框的颜色误认为是指针或安全区。
*   **Blue pixel trigger (蓝色像素触发阈值):** 在咬钩检测区域内，匹配到多少个目标颜色像素才判定为“鱼咬钩”。如果机器人经常过早提竿（误触），请调高此值；如果鱼咬钩了却没反应，请调低此值。

## 3. Timing (时间控制)
定义了机器人状态机的各类延时和超时设置，确保逻辑流畅且不会卡死。

*   **Cast animation (s) (抛竿动画时间):** 按下抛竿键后，等待多久才开始检测咬钩。如果在抛竿动画结束前就开始检测，可能会引发误判。
*   **Bite timeout (s) (咬钩超时):** 抛竿后，最多等待多久如果没有鱼咬钩就强制收竿并重新抛竿。
*   **Lost frame limit (丢失帧数上限):** 在拉扯阶段，连续多少帧找不到指针或安全区时，判定为拉扯结束（鱼脱钩或已捕获）。
*   **Result wait (s) (结算等待时间):** 成功捕获鱼后，等待多久再去关闭结算界面。如果关闭得太快导致游戏没反应，可以适当延长。
*   **Waiting poll (s) (等待轮询间隔):** 等待咬钩时，检测屏幕的频率。
*   **Tracking poll (s) (追踪轮询间隔):** 拉扯鱼时，更新 PID 计算和图像识别的频率。越低反应越快，但也更消耗 CPU。
*   **Bait error limit (鱼饵耗尽错误限制):** 连续多少次抛竿后立即弹出错误对话框（通常是因为鱼饵耗尽）。达到此限制后机器人会自动停止，防止无限死循环。
*   **Max struggle (s) (最大拉扯时间):** 拉扯阶段的安全超时时间（默认 120 秒）。超过此时间强制结束当前钓鱼动作，防止异常卡死。

## 4. Input & Hotkeys (输入与快捷键)
管理游戏内的按键映射以及机器人的全局控制快捷键。

*   **Key Bindings (按键绑定):**
    *   **Cast key:** 抛竿/提竿键（默认 F）
    *   **Move left/right:** 拉扯时的左右移动键（默认 A 和 D）
    *   **Exit key:** 关闭界面的按键（默认 Esc）
*   **Result close (结算关闭方式):** 捕获鱼后如何关闭结算弹窗。可选“Click center”（点击屏幕中心）或“Press exit key”（按下设定的 Exit 键）。
*   **Always on top (总在最前):** 让机器人的控制面板始终悬浮在游戏窗口上方。
*   **Debug logging (调试日志):** 开启后会将详细的 PID 追踪数据写入 `fishing_data.csv`，仅在排查问题时建议开启。
*   **Monitor (显示器):** 如果您有多个显示器，请选择运行游戏的那一个，以确保画面捕获区域正确。
*   **Global Hotkeys (全局快捷键):**
    *   **Toggle:** 暂停/恢复机器人的快捷键（默认 F8）。建议在需要手动操作菜单时按下暂停。
    *   **Stop:** 完全停止机器人的快捷键（默认 F12）。


## 5. Humanization (拟人化 - 防封检测)
引入随机性和不完美的反应，使机器人的操作看起来更像真实玩家，降低被游戏反作弊系统检测的风险。

*   **Humanize input (启用拟人化):** 总开关。建议始终开启。
*   **Key Pulse Timing (按键脉冲时间):**
    *   **Pulse hold min/max:** 在拉扯阶段，按下按键的随机持续时间范围。
    *   **Pulse gap min/max:** 两次按键之间的随机停顿时间范围。
*   **Deadband Micro-corrections (死区微调):**
    *   **Deadband taps:** 开启后，当指针处于死区（完美对齐）时，也会偶尔发送极短的“微点”按键，模仿人类的手抖和无意识微调。
    *   **Tap chance:** 每个循环触发死区微调的概率。
*   **Reaction Latency (反应延迟):**
    *   **Reaction min/max:** 模拟人类的视觉反应时间。看到需要移动后，延迟这段时间才真正按下按键。
    *   **Reaction dist:** 随机延迟的分布模式（均匀分布 uniform、高斯聚集分布 gaussian、偏短时间分布 exponential）。
*   **PID Noise (PID 噪声):** 
    *   在计算出的完美 PID 输出上叠加一层随机噪声，使移动轨迹变得不那么机械和精准。
*   **Timing Jitter (时间抖动):** 
    *   给抛竿动画、结算等待时间等固定时间增加随机的浮动范围。
*   **Adaptive Focus (自适应专注):**
    *   模拟真实玩家的行为：当鱼跑得非常偏、快要脱钩时，玩家的注意力会高度集中，反应变快。开启此项后，误差越大，机器人设置的延迟就会自动变低，按键变得更频繁，以迅速把鱼拉回安全区。

## 6. System & Updates (系统与更新)

*   **Check for Updates:** 检查 GitHub 上是否有新版本的 NTE Auto-Fish 发布。如果有新版本，会显示“Download Update”按钮，点击将自动在浏览器中打开下载页面。

---

**遇到问题怎么办？**
1. 如果钓鱼条总是跟不上：检查 `PID Tuning` 中的 Kp 是否太低，或者 `Humanization` 中的反应延迟 (Reaction) 是否设置得过高。
2. 如果游戏到了夕阳/夜晚就不提竿：这通常是因为游戏内光线变暖，导致蓝色提示像素变少。请在 `Vision & Detection` 中适当降低 `Blue pixel trigger`，或者重新校准 `Safe Zone` 和 `Cursor` 的 HSV 范围。
3. 如果需要修改检测区域：可以在控制台或主界面触发 `recalibrate` 重新校准。
